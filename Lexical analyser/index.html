Theory

Understanding the fundamentals of Lexical Analysis

What is a Lexical Analyser?

A Lexical Analyser (also called a Lexer or Scanner) is the first phase of a compiler. It reads the input source code as a stream of characters and converts it into a sequence of tokens. Each token represents a meaningful unit such as a keyword, identifier, number, operator, or separator.


Key Functions

  • Reads the source program character by character
  • Groups characters into lexemes
  • Classifies each lexeme into a token type
  • Removes whitespace and comments
  • Reports lexical errors (e.g., invalid characters)
  • Tracks line numbers for error reporting

How It Works

The lexer uses pattern matching rules to identify tokens. For example, if the current character is a letter, it continues reading until a non-alphanumeric character is found, then checks if the resulting word is a keyword or an identifier. Similarly, digits are grouped to form numbers, and special symbols are recognized as operators or separators.

Token Types

Categories of tokens recognized by the Lexical Analyser

Token Type Description Examples
KEYWORD Reserved words of the language int, float, if, else, return, while, for, char, double, void
IDENTIFIER User-defined names for variables, functions main, sum, count, myVar
NUMBER Integer and floating-point constants 42, 3.14, 100, 0
STRING Text enclosed in double quotes "hello", "world"
OPERATOR Arithmetic, relational, and logical operators +, -, *, /, =, <, >, !, &, |
SEPARATOR Punctuation and grouping symbols ; , ( ) { } [ ]
UNKNOWN Unrecognized characters @, #, ~
